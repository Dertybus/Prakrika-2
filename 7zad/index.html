Коротко

z-index работает для позиционированных элементов (например, с position: relative/absolute/fixed). Чем больше число — тем выше слой в пределах одного контекста наложения.

overflow управляет тем, что делать с содержимым, выходящим за границы блока: hidden обрезает, scroll/auto добавляет прокрутку, visible показывает всё.

2) clamp()

clamp(min, preferred, max) помогает делать “плавный” адаптив: значение растёт/уменьшается, но не выходит за границы min/max. Попробуйте менять ширину окна браузера.

Заголовок с clamp()

Пример: font-size: clamp(18px, 1.6vw + 12px, 30px)

Размер меняется плавно И не выходит за заданные min/max.

Когда удобно использовать

- адаптивные шрифты и отступы без “скачков” на брейкпоинтах
- ширину/высоту карточек, когда нужно ограничить минимум/максимум
- любой параметр, который должен “дышать” от размера экрана

3) position — 4 разных блока

Ниже показаны static, relative, absolute (внутри родителя) и fixed. Блоки оформлены рамками/скруглением/тенью (как в пункте 7.1).

position: static (по умолчанию)

Элемент идёт в обычном потоке. top/right/bottom/left не влияют.

position: relative

Блок остаётся в потоке, но визуально смещён: top: 10px; left: 10px.

position: absolute

Вынут из потока и позиционируется относительно ближайшего “не static” родителя (этой рамки).

Родитель для absolute

У обёртки задано position: relative, поэтому абсолютный блок привязан к ней.

Доп. контент (для прокрутки)

Прокрутите страницу — справа снизу закреплён блок position: fixed.

position: fixed Этот блок закреплён внизу справа и не двигается при прокрутке.
