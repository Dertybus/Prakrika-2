Настоящий документ содержит руководство оператора для программы «Интерактивная энциклопедия игр "Steam Wiki"». В руководстве приведены сведения о назначении программы, условиях её выполнения, описаны основные функции и порядок работы пользователя (оператора), а также даны сообщения системы и рекомендации по действиям при их возникновении. Документ предназначен для пользователей программы, имеющих базовые навыки работы с веб‑браузером.

1. Назначение программы

Программа «Интерактивная энциклопедия игр "Steam Wiki"» предназначена для создания, хранения и коллективного редактирования статей, связанных с компьютерными играми, а также для организации обсуждений и взаимодействия пользователей.

Функциональное назначение

  • регистрация и аутентификация пользователей (в том числе через учётную запись Steam);
  • поиск статей по ключевым словам и фильтрация по категориям;
  • просмотр, комментирование и оценка статей (лайки/дизлайки);
  • создание новых статей и редактирование существующих (для пользователей с соответствующими правами);
  • модерация контента и управление правами пользователей (для модераторов и администраторов);
  • подписка на разделы и получение уведомлений об изменениях;
  • ведение личного кабинета с настройками профиля.

Эксплуатационное назначение

Программа эксплуатируется на серверной стороне и доступна пользователям через интернет с помощью стандартных веб‑браузеров. Пользователями программы являются зарегистрированные участники (читатели, редакторы, модераторы, администраторы).

Кратко об условиях выполнения

См. полные требования к технике и ПО в разделах 2.1–2.3 руководства.

  • ПК/мобильное устройство с не менее 2 Гбайт ОЗУ;
  • современный браузер (Chrome, Firefox, Safari, Edge 100+);
  • подключение к Интернету от 1 Мбит/с;
  • базовые навыки работы в сети Интернет.

Навигация Подробная схема интерфейса, сценарии входа, поиска, редактирования и модерации раскрываются в разделах ниже.

2. Условия выполнения программы

2.1. Технические средства

  • персональный компьютер или мобильное устройство;
  • процессор с тактовой частотой не менее 1,5 ГГц;
  • оперативная память не менее 2 Гбайт;
  • экран с разрешением не ниже 1024×768;
  • устройство ввода (клавиатура, мышь или сенсорный экран);
  • постоянное подключение к сети Интернет со скоростью не менее 1 Мбит/с.

2.2. Программные средства

  • операционная система Windows, macOS, Linux, Android или iOS;
  • современный веб‑браузер: Chrome, Firefox, Safari или Edge версии 100 и выше;
  • учётная запись Steam для входа через внешний сервис (по желанию).

2.3. Требования к персоналу

  • базовые навыки работы с веб‑браузером и сетью Интернет;
  • знание правил модерации и настоящего руководства для модераторов и администраторов;
  • специальная группа по электробезопасности не требуется.

3. Интерфейс пользователя

3.1. Главное окно программы

Главное окно системы представлено веб‑страницей с несколькими областями:

  • Шапка — логотип, поле поиска, кнопки «Войти» / «Регистрация», ссылка на личный кабинет;
  • Левая панель — навигация по категориям (игры, жанры, платформы) и основным разделам;
  • Центральная область — рекомендуемые и популярные статьи, новости проекта;
  • Правая панель — уведомления, блоки подписок и активные обсуждения.

3.2. Навигация по разделам

  • главное меню категорий и разделов;
  • ссылки внутри статей (переход по связанным материалам);
  • поисковая строка по ключевым словам.

3.3. Личный кабинет

После входа пользователю доступен личный кабинет со вкладками:

  • «Профиль» — личные данные, аватар, описание;
  • «Настройки» — тема, язык, уведомления;
  • «Мои подписки» — список подписок на статьи и разделы;
  • «Мои статьи» — созданные и редактируемые материалы;
  • «Уведомления» — история уведомлений от системы.
Схема навигации

Обобщённая схема переходов между основными экранами (см. рисунок 1 в задании).

  • Главная страница точка входа, поиск, навигация
  • Регистрация / вход учётная запись, права доступа
  • Просмотр статей чтение, оценка, обсуждение
  • Личный кабинет профиль, подписки, статьи
  • Модерация статьи, комментарии, права

4. Выполнение программы

4.1. Вход в программу

  1. 1Открыть главную страницу сайта в браузере.
  2. 2Нажать кнопку «Войти» в шапке.
  3. 3Выбрать способ входа: через Steam или по логину/паролю.
  4. 4После успешной аутентификации пользователь возвращается на главную страницу.

4.2. Поиск и просмотр статей

  1. 1Ввести ключевые слова в поле поиска (например, «Portal 2»).
  2. 2Нажать Enter или кнопку поиска.
  3. 3Выбрать нужную статью из списка результатов.
  4. 4На странице статьи доступны текст, мультимедиа, блок оценок и обсуждение.

4.3. Комментирование статей

  1. 1Прокрутить страницу до раздела «Обсуждение».
  2. 2Ввести текст комментария.
  3. 3Нажать «Отправить» и дождаться публикации.
  4. 4Для ответа использовать ссылку «Ответить» под нужным комментарием.

4.4. Редактирование и модерация

Создание и правка статей доступны пользователям с ролями «редактор», «модератор» или «администратор».

  • создание новой статьи через вкладку «Мои статьи» → «Создать статью» в личном кабинете;
  • редактирование существующей статьи по кнопке «Править» на странице материала;
  • модерация статей и комментариев в специальном разделе «Модерация»;
  • управление правами пользователей (смена ролей) — только для администратора.

4.5. Завершение работы

Для завершения работы достаточно закрыть вкладку браузера или выполнить явный выход через меню с именем пользователя («Выйти»).

5. Сообщения оператору

5.1. Сообщения об успешных действиях

Сообщение Описание Действия оператора
«Вы успешно зарегистрировались. Проверьте почту для подтверждения.» Появляется после отправки формы регистрации. Перейти в почту и подтвердить адрес.
«Вход выполнен» Отображается после успешной аутентификации. Продолжить работу в системе.
«Статья сохранена» Результат сохранения изменений в статье. Дополнительных действий не требуется.
«Комментарий добавлен» Сообщение после отправки комментария. При необходимости отредактировать или удалить.
«Статья одобрена» Уведомление модератору об успешной публикации. Информирование, дальнейших действий не требуется.

5.2. Сообщения об ошибках

Сообщение Описание Действия оператора
«Ошибка входа: неверный логин или пароль» Введены некорректные учётные данные. Проверить ввод, при необходимости восстановить пароль.
«Недостаточно прав для выполнения действия» Попытка выполнить операцию без соответствующей роли. Обратиться к администратору для изменения прав.
«Статья с таким названием уже существует» Дублирование заголовка при создании статьи. Изменить название статьи.
«Ошибка соединения с сервером» Проблемы с интернетом или сервером приложения. Проверить подключение и повторить попытку позже.
«Не удалось загрузить изображение. Допустимые форматы: JPG, PNG, GIF» Неподдерживаемый формат или повреждённый файл. Выбрать файл в допустимом формате.

5.3. Действия при сбоях

  • обновить страницу (F5 или кнопка обновления в браузере);
  • очистить кэш браузера и повторить попытку;
  • проверить стабильность интернет‑соединения;
  • при сохранении проблемы — обратиться к администратору через форму обратной связи.
