ТЕХНИЧЕСКОЕ ЗАДАНИЕ 
1. Цель работы
Закрепить навыки семантической верстки, стилизации с помощью CSS (Flexbox/Grid), обеспечения адаптивности (десктоп → мобильные) и написания простых скриптов на чистом JavaScript.
2. Описание проекта
«RepairService» — Сайт сервисного центра по ремонту техники
•	Описание: Ремонт телефонов, ноутбуков, планшетов, прайс-лист.
•	Особенности: На главной — таблица цен на популярные услуги. В каталоге — список неисправностей с ценами.
•	Дополнительный JS: Калькулятор стоимости ремонта (выбрать устройство и тип поломки → итоговая цена).
•	Цветовая схема: Светло-серый фон #F5F5F5, акцентный синий #1976D2, темно-серый текст #424242
•	Шрифты: Roboto Condensed (заголовки) + Open Sans (текст)

Необходимо сверстать сайт для вымышленной компании
Сайт должен состоять из 6 HTML-страниц:
1.	index.html — Главная (промо-страница с преимуществами и товарами).
2.	catalog.html — Каталог товаров (сетка карточек с фильтрацией по категориям).
3.	product.html — Карточка товара (детальное описание конкретного продукта).
4.	blog.html — Блог (список статей/новостей).
5.	article.html — Страница отдельной статьи (пример текстовой страницы с контентом).
6.	contacts.html — Контакты (страница с формой обратной связи и картой).
3. Требования к структуре (общий макет)
На всех страницах должны присутствовать общие блоки:
•	Шапка (Header): Логотип с гиперссылкой на главную страницу, навигационное меню с подвкладками (Главная, Каталог, Блог, Контакты), иконка корзины (счетчик товаров — можно заглушка), поисковая строка (упрощенная, без функционала).
•	Основной контент (Main): Уникальное содержимое для каждой страницы.
•	Подвал (Footer): Копирайт (© 2026 RepairService. Учебный проект, ФИО разработчика), иконки соцсетей (можно просто картинки-ссылки), ссылки на основные разделы (дубль меню), короткая информация о компании.
4. Требования к технологиям (Стек)
•	HTML: Семантическая верстка (<header>, <main>, <section>, <article>, <footer>). Наличие валидного кода (исходный код, написанный в строгом соответствии со стандартами и правилами, установленными консорциумом W3C. Такой код не содержит синтаксических ошибок, корректно использует теги, обеспечивает хорошую кроссбраузерность).
•	CSS:
o	Использование Flexbox для выравнивания элементов в шапке и в карточках.
o	Использование Grid для построения сетки карточек в каталоге и блоге.
o	Адаптивность через медиазапросы (смотри пункт 8).
o	Запрещено использовать готовые библиотеки (Bootstrap, Tailwind) — только чистый CSS.
•	JavaScript:
o	Чистый JS (Vanilla JS).
o	Без подключения jQuery.
o	Весь JS должен находиться в отдельном файле script.js.
5. Описание страниц
5.1. Главная страница (index.html)
1.	Блок 1 (Hero/Приветствие):
o	Заголовок (H1): "RepairService".
o	Подзаголовок: "Починим все".
o	Кнопка "Перейти в каталог" (ссылка на catalog.html).
2.	Блок 2 (Преимущества):
o	Заголовок H2: "Почему выбирают нас".
o	Сетка из 3-4 преимуществ (иконка + заголовок + текст).
3.	Блок 3 (Популярные услуги):
o	Заголовок H2: "Популярные услуги".
o	Сетка из 3 карточек товаров.
o	Содержимое карточки:
	Изображение товара.
	Название товара.
	Цена.
	Кнопка "Подробнее" (ссылка на product.html).
	Кнопка "В корзину".
	Кнопка «Смотреть больше» (переход в каталог).
4.	Блок 4 (Акция/Баннер):
o	Промо-баннер с призывом подписаться на рассылку и получить скидку.
o	Поле ввода email и кнопка "Подписаться" (без реальной отправки, просто демонстрация).
5.2. Страница каталога (catalog.html)
1.	Заголовок H1: "Каталог товаров".
2.	Боковая панель (aside): Фильтр "Категории" со списком ссылок (ПК, телефон, планшет, домашняя техника). При клике на ссылку должно меняться содержимое страницы (учебная задача: можно просто подсвечивать активную категорию или показывать сообщение в консоли).
3.	Сетка каталога:
o	Использовать CSS Grid для создания колонок.
o	Минимум 4 карточек товаров (для наглядности сетки).
o	Карточки аналогичны главной странице (название + цена + кнопки), но с добавлением краткого описания (например, "Вес: 500г").
5.3. Страница товара (product.html)
1.	Заголовок H1: (Название конкретного товара, например, "Восстановление данных с SDD").
2.	Блок с деталями:
o	Крупное изображение товара.
o	Подробное описание (состав, страна происхождения, способ применения).
o	Цена.
o	Кнопка "Добавить в корзину".
o	Дополнительно: Вкладки (табы) с информацией: "Описание", "Характеристики", "Отзывы" (можно реализовать через слайдеры).
5.4. Страница блога (blog.html)
1.	Заголовок H1: "Советы по уходу за техникой".
2.	Сетка статей (Grid):
o	Минимум 4 карточки статей.
o	Содержимое карточки статьи:
	Изображение-превью.
	Заголовок статьи.
	Дата публикации.
	Краткий анонс (1-2 предложения).
	Ссылка "Читать далее" (ведет на article.html).
5.5. Страница статьи (article.html)
1.	Заголовок H1: "Как чистить ПК от пыли" (или любая другая тема).
2.	Метаинформация: Дата публикации, автор (можно вымышленный).
3.	Текстовый блок: 2-3 абзаца текста с подзаголовками H2.
4.	Изображение: Картинка с подписью (вставлена в текст).
5.	Блок "Похожие статьи": Ссылки на 2-3 другие статьи (внизу страницы).
5.6. Страница контактов (contacts.html) — дополнительно
1.	Заголовок H1: "Контакты".
2.	Контактная информация: Адрес, телефон, email (оформлено списком).
3.	Форма обратной связи: Поля "Имя", "Email", "Сообщение", кнопка "Отправить" (без реальной отправки, при клике — alert "Сообщение отправлено").
4.	Карта: Встроенная карта-схема проезда (можно скриншот или iframe от Яндекс/Google Карт).
6. Требования к дизайну (примерная стилизация)
Дизайн должен быть простым, но аккуратным, все страницы сайта должны иметь связанный контент по выбранной теме с элементами продуманного веб-дизайна.
•	Состояние ховера: Все кликабельные элементы (ссылки, кнопки) должны менять цвет или прозрачность при наведении мыши. Карточки товаров могут немного приподниматься.
7. Навигация и меню
•	В шапке должно быть реализовано "липкое" меню (sticky header) — при прокрутке страницы шапка остается вверху.
•	Пункт меню, соответствующий текущей странице, должен быть активным (подсвечен другим цветом или подчеркиванием).
8. Требования к адаптивности (Отзывчивость)
Верстка должна корректно смотреваться на трех контрольных точках:
1.	Десктоп (ширина > 1024px): Горизонтальное меню, 3-4 колонки в каталоге/блоге, боковая панель видна.
2.	Планшет (от 768px до 1023px): 2 колонки в каталоге/блоге, боковая панель можно скрыть или перенести вниз. Размер шрифта можно чуть уменьшить.
3.	Телефон (< 767px):
o	1 колонка в каталоге/блоге (карточки идут друг за другом).
o	Боковая панель скрыта или перенесена в самый низ.
o	Меню в шапке должно превращаться в "выпадающее" (бургер-меню) — обязательное требование.
o	Поиск и иконка корзины могут быть скрыты в бургер или оставаться в шапке в уменьшенном виде.
9. Функциональность JavaScript (Интерактив)
В файле script.js должен быть реализован следующий функционал:
1.	Работа с корзиной (учебная):
o	Найти все кнопки "В корзину" на странице (index.html, catalog.html, product.html).
o	При клике на любую такую кнопку:
	В консоль браузера выводится сообщение: "Товар [НАЗВАНИЕ ТОВАРА] добавлен".
	Показывается всплывающее окно alert (как указано в п. 5.1).
	Счетчик товаров рядом с иконкой корзины увеличивается на 1 (хранить значение в переменной).
2.	Текущий год в подвале:
o	Найти элемент в футере (например, спам с классом current-year).
o	Автоматически подставить в него текущий год (2026, 2027...), используя объект new Date().
3.	Бургер-меню (для мобильной версии):
o	Создать кнопку-бургер (три полоски), которая видна только на мобильных экранах.
o	При клике на бургер, меню навигации должно плавно появляться/исчезать.
4.	Табы на странице товара:
o	Реализовать переключение между вкладками "Описание", "Характеристики", "Отзывы" без перезагрузки страницы (скрывать/показывать блоки).
12. Дополнительные задания (для тех, кто делает быстро)
•	Добавить плавную прокрутку (scroll-behavior: smooth) к якорям.
•	Сделать затемнение или увеличение картинок в карточках при наведении.
•	Реализовать простую "фильтрацию" товаров на каталоге при клике на категории (скрывать карточки других категорий через добавление класса hidden).
•	Добавить "карусель" (слайдер) на главной странице в блоке с популярными товарами (можно ручной слайдер на чистом JS).
